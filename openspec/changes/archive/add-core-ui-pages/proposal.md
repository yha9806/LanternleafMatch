# Proposal: Add Core UI Pages

## Change ID
`add-core-ui-pages`

## Summary

添加游戏核心 UI 页面系统，包括主菜单（首页）、关卡选择、设置页面，形成完整的游戏导航闭环。

## Why

### 问题
- 当前只有一个 Demo 页面，无法形成完整游戏流程
- Menu.scene 和 Game.scene 存在但内容为空
- 用户无法选择关卡、调整设置、查看进度
- 无法发布为正式产品

### 收益
- 完整的游戏导航流程
- 用户可以选择任意已解锁关卡
- 提供设置选项（音量、震动等）
- 为后续功能（收藏、排行榜）奠定基础

## What

### 新增能力

1. **主菜单 (Main Menu)**
   - 游戏标题和品牌展示
   - 开始游戏按钮
   - 进入设置按钮
   - 显示玩家进度（当前关卡）
   - 体力状态显示

2. **关卡选择 (Level Select)**
   - 50 关卡地图展示
   - 已解锁/未解锁状态
   - 星级评价显示
   - 当前关卡高亮
   - 滚动浏览支持

3. **设置页面 (Settings)**
   - 背景音乐开关/音量
   - 音效开关/音量
   - 震动反馈开关
   - 语言选择（预留）
   - 重置进度（需确认）

### 页面流转

```
[主菜单]
    │
    ├── 开始游戏 → [关卡选择] → 选择关卡 → [游戏场景]
    │                              │
    │                              └── 返回 → [主菜单]
    │
    └── 设置 → [设置页面] → 返回 → [主菜单]

[游戏场景]
    │
    ├── 暂停 → 返回主菜单 → [主菜单]
    │
    ├── 通关 → 下一关 → [游戏场景]
    │       └── 返回关卡选择 → [关卡选择]
    │
    └── 失败 → 重试 → [游戏场景]
            └── 返回 → [关卡选择]
```

## Impact

### 影响范围
- 新增 3 个 UI 组件
- 新增场景管理器
- 新增玩家进度存储
- 修改 Menu.scene 和 Game.scene

### 不影响
- 核心游戏逻辑
- 现有弹窗系统
- 平台适配层
- E2E 测试系统

## Success Criteria

- [ ] 主菜单正确显示玩家进度
- [ ] 关卡选择展示 50 关，正确显示解锁状态
- [ ] 设置页面可保存并持久化配置
- [ ] 页面间导航流畅，无闪烁
- [ ] 响应式适配竖屏 1080×1920
- [ ] E2E 测试覆盖核心流程
